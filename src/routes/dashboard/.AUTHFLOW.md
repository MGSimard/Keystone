# Dashboard/.AUTHFLOW.md

Notes on authentication flow.

- No dedicated sign-in/sign-up pages
- Intercept dashboard route access, render auth flow if not authenticated
- Domain-based authentication (email)
- Uses Cognito + company IdP (e.g., AWS IAM Identity Center)

## Auth Flow

Elements:

- OAuth Button
- Separator
- "Have an invite?" type CTA under

Conditionals:

AUTHED + AUTHORIZED -> Dashboard
UNAUTHED or UNAUTHORIZED -> Auth component

--

UNAUTHED:

1. Click OAuth button, redirect process
2. Authenticate
3. Return to /dashboard route to and run conditionals again

AUTHED:

1. Check if user email has an existing organization under that domain

- Yes: Are they currently permitted in the organization?
  - Yes: View dashboard
  - No: Notify user, give option to apply for access
- No: Create new organization (#2 below)

2. Check if user has IdP permission to create an org on their company's behalf(?)

- Yes: Onboarding, organization registration flow
- No: Notify user, give instructions on how to apply permissions in their IdP
